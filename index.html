Enunciado del problema

La empresa ficticia Transous tiene un sistema de base de datos en SQL Server donde guardan la informacion del ingreso del personal a cada una de sus sedes, esta tabla esta compuesta por ID(cedula del empleado), NAMEIN(Nombre del empleado), EMAIL(que es el correo corporativo del empleado), DATOS(la sede y al edificio al que ingreso), CIUDAD(la ciudad donde se hubica la sede), Permiso(El grado del permiso que tuvo al entrar junto con su identificador) y FECHAINGRE(la fecha en la que ingreso a la instalacion), en los lineamientos del area de tecnologia esta la meta de migrar esa base de datos a ElasticSearch. Su sistema en SQL tiene una serie de requerimientos a resolver que seran resultos durante el traspaso usando LogStash

Requerimientos:

  • En el sistema de SQL, por una falla del sistema las horas de ingreso quedaron guardadas 5 horas antes de la hora correcta (GTM -5)
  • El nuevo sistema en ElasticSearch debe tener un field que contenga el usuario empresarial que es el mismo del correo sin el dominio
  • Los nombres de las columnas deben ser cambiados: ID = Cedula, NAMEIN = Nombre, EMAIL = Email, DATOS = Sede, CIUDAD = Ciudad, PERMISO = PERMISO, FECHAINGRE = FechaDeIngreso
  • Se requiere un field que contenga el numero de la sede y la letra del edificio que se llame SedeID
  • Se requiere un field que contenga el ID del acceso llado IDacceso y otro que contenga el nivel de acceso otorgado llamado NIVELacceso

Adicionalmente Transous tiene un script de alertamiento para saber cuando hay un acceso no autorizado, este script lee constantemente un csv en el formato especificado de los requerimentos, las operaciones de lectura de la base de datos y de generación del reporte deben ser ejecutadas en paralelo y por el mismo Logstash
